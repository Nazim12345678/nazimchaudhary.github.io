Global Dispenser IoT Analytics Platform

Client: The Coca-Cola Company (Global) Partner: Thought Logic Role: Data Architect & Lead Implementation

Overview

Implemented a real-time IoT data processing platform for Coca-Cola's Freestyle dispenser network using Microsoft Fabric Eventhouse (KQL Database). The solution processes telemetry data from 55,000+ dispensers globally, handling 5.28 million files daily with sub-second query performance.

Business Challenge

Coca-Cola's global Freestyle dispenser network generates massive volumes of operational data requiring both real-time monitoring and historical analysis:

  • 220,000 files per hour (5.28M files daily) from 55,000 dispensers
  • 165 GB compressed daily data (~660 GB uncompressed)
  • Need for real-time operational dashboards and historical trend analysis
  • Multiple consumer teams requiring different data access patterns
  • Compliance requirements for 2-year data retention
  • Cost optimization through intelligent data tier routing

Solution Architecture

Data Ingestion

Dual-path ingestion strategy supporting both streaming and batch processing:

  • Streaming Path: Fabric Event Streams for real-time telemetry ingestion
  • Batch Path: Azure Blob Storage landing zone with automated file triggers
  • Orchestration: Fabric Pipeline Auto Execution with Reflex triggers

Medallion Architecture in Eventhouse

Bronze Kusto DB (Raw Layer)

  • Raw table ingestion from gzip files (DeviceStyle OS, UI, Java Logs)
  • Update Policy for streaming data capture
  • Transformation Function for initial parsing and validation
  • Complete audit trail and lineage tracking

Silver Kusto DB (Refined Layer)

  • Parse and validate log structures
  • Schema standardization across dispenser types
  • Deduplication and data quality checks
  • 2-year rolling retention policy
  • Optimized for recent historical queries (hours to days)

Gold Kusto DB (Real-time Analytics)

  • Curated views for operational monitoring
  • Pre-aggregated metrics for dashboard performance
  • Business semantic layer with friendly naming
  • Agent-ready data structures for intelligent routing
  • Sub-second query response times

Gold DWH (Batch Analytics)

  • Fabric Data Warehouse for complex historical analysis
  • Daily batch loads from orchestration layer
  • Advanced aggregations and dimensional modeling
  • Long-term storage optimization with compression
  • Integration with Fabric OneLake for unified data access

Intelligent Query Routing

Data Agent analyzes incoming queries and routes to optimal storage tier:

  • Real-time operations: Kusto Gold DB (operational dashboards, live monitoring)
  • Recent historical: Kusto Silver DB (last few days analysis)
  • Long-term analytics: Fabric DWH Gold (trend analysis, compliance reporting)

Hybrid Gold Layer: Kusto for Real-time + DWH for Batch Analytics

Technical Specifications

  • Infrastructure: Fabric Capacity F128 SKU (F256 recommended for production)
  • Processing: Incremental loads (daily new data only)
  • Query Language: KQL (Kusto Query Language) for Eventhouse
  • Data Retention: 2-year rolling window in Silver layer
  • Performance Target: Sub-second query response for operational dashboards
  • Availability: 99.9% uptime SLA for streaming pipeline

Analytics & Serving

  • SQL Endpoint: Direct Query access to Fabric Data Warehouse
  • Data Agent: Intelligent routing layer for query optimization
  • Power BI: Real-time dashboards with Direct Query and Import modes
  • Consumer Teams:
    • GEP Team - Global equipment performance monitoring
    • NAOU Team - North America operations tracking
    • Analytics Team - Strategic insights and reporting
